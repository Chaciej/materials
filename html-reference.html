Witaj - tutaj dokładnie nauczę cię jak pisać HTMLa i łączyć go z CSS (językiem stylów)

HTML jest tylko szkieletem strony, sam w sobie niewiele może ale dzięki CSS będzie ładnie wyglądał a dzięki Javascript dodatkowo robił magiczne rzeczy.

Jedyne co muszisz wiedzieć o HTMLu to znać kilka podstawowych elementów, inaczej tagów HTMLa lub na polskie "znaczników" znaczących fragmenty strony jako coś konkretnego np. to jest akapit a tam będzie lista. Zacznijmy od tego jak używać tagów

<element> treść </element>

Jak widzicie element składa się z 2 bloków - otwierającego <coś> i domykającego </coś> ze "slashem" / tuż przed nazwą ale po <

"element" w powyższym przykładzie to nazwa elementu, oczywiście nie ma takiego znacznika w HTMLu jak "element" ale istnieje cała lista użytecznych elementów z jakich będziemy korzystać, oto niektóre z nich:

<h1>

heading - inaczej nagłówek lub tytuł, można używać od h1 do h6, im dalej tym staje się mniej ważny np. h3 jest mniej ważny od h1 i domyślnie też wyświetli się mniejszy

<h1> Tytuł główny </h1>

<h2> Podtytuł </h2>

To jest tekst h3

A tutaj h1

Sprawdź w konsoli jak nie wierzysz (F12 lub Ctrl+Shift+C i namierz element)

<p>

paragraph - inaczej akapit, czyli blok tekstu pisanego, który zawiera pewną myśl, oddzielający inne fragmenty tekstu dla lepszego poukładania

<p> Lorem ipsum dolor sit amet </p>

<p> Consectetur adipisicing elit </p>

Bla bla... coś tam lala, oto jest pierwszy akapit (eng. paragraph). Akapit może zawierać tyle tekstu, ile tu napchasz, pamiętaj jednak, że powinien służyć zarówno czytelności jak i merytorycznie rodzielać fragmenty różnych myśli tekstu.

Lorem ipsum dolor sit amet, consectetur adipisicing elit. Ad consectetur consequatur deleniti, deserunt dolorem excepturi expedita labore laudantium, placeat quod tempora tempore totam. Consectetur esse exercitationem fugit quae tenetur voluptates.

Akapity automatycznie dostają margines co sprawia, że wyglądają czytelniej, to całkiem cool, choć możecie im je zabrać za pomocą CSS, albo powiększyć nawet bardziej niż h1, pytanie tylko po co 🤔

Kolejny przykład (to też jest akapit, czyli p)

Lorem ipsum dolor sit amet, consectetur adipisicing elit. Accusantium animi consectetur doloremque doloribus eius, excepturi expedita fugit laborum nostrum nulla odit possimus quam quasi quod sit, tenetur unde vero voluptatum.

<div>

tzw. division - od wycinek

Znacznik div znaczy... cokolwiek, dlatego będzie jednym z najczęsciej używanych elementów, służących zwykle do wydzielania fragmentów strony i grupowania treści

W HTMLu powszechnym jest wkładanie w siebie wielu div'ów i w ten sposób zagnieżdżanie elementów (potem możemy taką grupkę np. przenieść w inne miejsce, wyrównać do lewej i każdy element w parencie będzie przemieszczał się razem z nim)

<span>

Span jest również wycinkiem ale bardziej tekstu niż bloku strony. Użyjemy go do oznaczania fragmentów akapitu np. do podświetlania poszczególnych partii tekstu.

Różnica między div a span jest taka, że div wyświetla się jako "block" a span jest "inline", tutaj przykład w praktyce:

Elementy typu "block"
To jest pierwszy div
To jest drugi div
To jest trzeci div

Elementy typu "inline"
To jest pierwszy span To jest drugi span To jest trzeci span

Jak widzimy każdy div wyświetlił się w nowej linii a span dalej płynął niczym tekst

Przykładowe elementy block: div, h1, p, hr
Przykładowe elementy inline: span, img, button, input, a, li, strong

Czas na atrybuty

Atrybuty siedzą wewnątrz elementu i nadają mu dodatkowe cechy
Poznajmy przy okazji 2 nowe tagi:


<img>

Oto jest obrazek (image)

<a>

A tutaj jest link (tzw. anchor)

Dwa powyższe tagi wymagają atrybutu, żeby w ogóle działać!

Img wymaga atrybutu src - inaczej source czyli adres do obrazka
zaś a wymaga href - adresu do strony, na który udamy się po kliknięciu linka ("href" znaczy tutaj http reference)

Zapiszemy je w następujący sposób:

<img src="adres">

<a href="adres"> kliknij mnie </a>

Jak widzicie wartość atrybutu jest zawsze w cudzysłowiu np. atrybut="wartość"

<img src="obrazki/moje/ja-na-plazy.jpg">

Powyższy adres próbuje wczytać obrazek bezpośrednio z dysku

Zauważcie że spacje są zastąpione myślnikami i raczej nie chcemy używać polskich znaków w nazwach plików

<img src="https://superwarsztaty.pl/sw-main/images/kroung.jpg">

Jeśli w adresie jest https:// to prawdopodobnie wczytujemy go z serwera w internecie

Możecie skopiować adres do dowolnego obrazka w internecie klikając na niego prawym a następnie "kopiuj adres obrazka" lub w angielskiej wersji "copy image address"

W HTMLu wygląda to tak:

<a href="#futer">Leć na koniec strony!</a>

<a href="#dupa">Leć do elementu o ID = dupa</a>

Podejrzyj sobie w konsoli, nie przejmuj się klasami, służą jedynie dekoracji

Znak # to ID elementu, do którego chcemy się przenieść i siup! Oczywiście trzeba wcześniej taki element sprecyzować:

<div id="przenieś-tutaj"> Treść elementu </div>
a potem < href="#przenieś-tutaj" a> Idziemy </a>

W CSS możemy dodatkowo sprecyzować dla elementu html scroll-behavior: smooth; dla płynniejszego scrollowania strony (domyślne zachowanie to natychmiastowy przeskok)

Istnieją pewne uniwersalne atrybuty

Uniwersalne znaczy w tym przypadku, że każdy element może je posiadać i będą znaczyć dokładnie tyle samo, np. każdy może mieć swój identyfikator czy klasę

Są to kolejno: id, class, title, lang, style

  • id: unikalny identyfikator elementu
  • class: klasa do której obiekt może należeć
  • title: każdy element może posiadać tytuł (wyświetla się po najechaniu)
  • lang: język tekstu w elemencie, przydatny dla Gugula
  • style: wewnątrz niego piszemy CSS dla elementu, tzw. inline style
<div id="zugubuludu">
Tylko 1 element może mieć to ID! Odtąd możesz robić z tym elementem następujące rzeczy:
  • Przewijać do niego tagiem <a> href="#zugubuludu"Idź tam </a> (jak w indeksie)
  • Namierzać go w Javascript po nazwie ID bez deklarowania zmiennej!
  • Stylować go w CSS za pomocą #zugubuludu {style} (# oznacza tutaj ID selector, tak jak . oznacza klasę) - UWAGA, nie jest to polecane o czym opowiem później

<div class="czarny fajny do-lewej">

Być może ciężko w to uwierzyć, ale powyższy element posiada aż 3 klasy CSS, są to: czarny, fajny i do-lewej, kolejność nie ma tutaj znaczenia

Możecie im teraz nadać dowolny wygląd w CSS stylując po klasie: .nazwa-klasy {nasze style} (pamiętajcie o kropce, to selektor klasy, do tego pamiętajcie o średnikach rozdzielających osobne linijki)

Z klasą jest trochę lepiej niż z ID bo 1 element może ich posiadać nawet kilka i nie ma limitu elementów posiadających konkretną klasę. Klasy możecie stylować w CSS i łapać w zmienne JS

<div class="przycisk editable">

Ten przycisk będzie posiadał własności zarówno klasy przycisk jak i editable

Inne opcjonalne atrybuty

  • name - przydatny w formularzach i inputach, żeby wyznaczyć do której grupy należą i dać możliwość np. podświetlenia się okna po kliknięciu odpowiedniej rubryki (label element)
  • width i height - działa w obrazkach i ustawia rozmiar (szerokość lub wysokość) na podaną wartość, nie potrzebuje jednostek px, domyślnie je ustawi (zalecane jest jednak użycie CSS do ustalania rozmiaru)
  • tabindex - określa czy element może być zaznaczony tzw. "focused" i sekwencjonowany klawiszem "Tab" - daj 0 jeśli tego właśnie chcesz, -1 jak chcesz by był zaznaczable ale nie sekwencjonable (przyciski są domyślnie ale np. nie div'y)
